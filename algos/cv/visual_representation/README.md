# 视觉表征学习 面试攻略

## 专题顺序

- 第一篇：[视觉表征学习](README.md)
- 第二篇：[CLIP](clip/README.md)
- 第三篇：[SigLIP](siglip/README.md)

## 这是什么？

视觉表征学习，讲的不是“图像分到固定类别里”这件事本身，而是：

- 怎么把一张图编码成可复用的向量表示
- 这个表示能不能迁移到检索、分类、聚类、VLM 等不同任务
- 表示里保留下来的，到底是局部纹理，还是高层语义

如果只用一句话概括：

> 视觉表征学习的核心目标，是学到一个可迁移、可比较、可复用的视觉 embedding，而不是只学会某个封闭类别集上的分类头。

## 核心机制

### 1. 为什么要单独讲“表征学习”？

因为很多视觉系统真正复用的，不是最后那个分类器，而是中间的 embedding。

例如：

- zero-shot 分类：拿图像 embedding 和文本 embedding 比相似度
- 图文检索：看图像和文本是不是在同一语义空间里接近
- 聚类 / 召回 / 去重：直接拿 embedding 做近邻检索
- VLM：把视觉 encoder 学到的表示接到 LLM 前面

所以从工程视角看，表征学习的价值在于：

> 一次训练好的视觉 backbone，可以给多个下游任务复用。

### 2. 它和传统图像分类的差别是什么？

传统图像分类更像：

```text
image -> backbone -> classifier head -> fixed labels
```

视觉表征学习更关心：

```text
image -> encoder -> embedding
embedding -> similarity / retrieval / downstream head
```

差异的关键在于：

- 分类头是任务定制的
- embedding 是希望跨任务复用的

### 3. 这个方向里最常见的路线有哪些？

面试里最常见的，通常是这两条：

#### 1. 单模态表征学习

例如自监督视觉表示、对比学习视觉表示。

重点是：

- 不依赖固定类别标签
- 学一个通用视觉空间

#### 2. 图文联合表征学习

最典型的就是 CLIP / SigLIP。

重点是：

- 图像和文本进入同一个语义空间
- 分类、检索、开放词表识别都能统一成“相似度匹配”

这也是为什么这个仓库目前先补 CLIP 和 SigLIP。

### 4. 视觉表征学习适合做什么？

- zero-shot 分类
- 图文检索
- 开放词表识别
- 召回、聚类、去重
- 作为 VLM 的视觉 encoder 底座

### 5. 它不太擅长什么？

只靠全局 embedding，一般不够解决这些问题：

- 精细 OCR
- 精确 grounding
- 目标检测 / 分割
- 细粒度计数
- 需要强空间推理的任务

原因也很直接：

> 这类方法主要学的是“整图级语义表示”，不是密集预测或区域级监督。

## 面试高频问题

### 1. 为什么视觉表征学习比分类头更值得复用？

因为 embedding 能迁移到更多任务，而分类头通常只服务一个封闭标签集。

### 2. 为什么 CLIP / SigLIP 经常被归到视觉表征学习里？

因为它们本质上都在学习图像和文本共享的语义 embedding 空间。

### 3. 为什么表征学习能支撑 VLM？

因为 VLM 前面的 vision encoder，本质上就是先要学到一个足够强的视觉表示，再交给 LLM 消费。

## 面试时怎么讲

比较稳的一种讲法是：

> 视觉表征学习关注的是 encoder 输出的 embedding 能不能跨任务复用，而不是某个分类头本身。对大模型时代尤其重要，因为 CLIP / SigLIP 这类图文表征模型不仅能做 zero-shot 分类和检索，还经常直接作为 VLM 的视觉前端。
