# 数据获取 / 采样 / 去重 面试攻略

## 这是什么？

这是数据工程部分的第一篇，先讲数据闭环最靠前的一段：

- 数据从哪来
- 采样为什么不是“越多越好”
- 去重为什么会直接影响模型效果和评测可信度

如果只用一句话概括：

> 数据获取的重点不是单纯把量堆大，而是按任务分布拿到足够有代表性的样本，再把重复、污染和明显低价值样本清掉。

## 核心机制

### 1. 数据获取在解决什么问题？

模型效果不好，不一定先怪模型结构，很多时候先要看：

- 数据源覆盖够不够
- 长尾场景有没有
- 不同来源分布是否严重失衡
- 训练集和评测集有没有泄漏

所以数据获取的目标不是“抓更多”，而是：

> 围绕目标任务构建一个覆盖合理、分布可控的数据池。

### 2. 为什么采样不是越均匀越好？

采样要看任务目标。

例如：

- 预训练数据往往更看重广覆盖
- 垂类任务更看重目标分布匹配
- 安全、审核、长尾识别更需要 hard case 过采样

所以采样策略通常至少会考虑：

- 来源比例
- 类别比例
- 长尾场景补偿
- 时间新鲜度

### 3. 为什么去重很重要？

去重至少影响三件事：

#### 1. 训练效率

重复样本太多，会浪费训练预算。

#### 2. 泛化能力

重复数据会让模型以为某些模式出现得特别频繁，带来分布偏置。

#### 3. 评测可信度

如果训练集和评测集泄漏，离线指标会被高估。

所以去重不是“锦上添花”，而是：

> 数据工程里最基本的质量底线。

### 4. 去重一般怎么做？

最基础可以分两层：

#### 1. exact dedup

完全相同的文本、图片、样本 id 直接去掉。

#### 2. near dedup

处理几乎重复的样本，例如：

- 改了几处标点的文本
- 同一张图不同分辨率版本
- 同一视频截出的高度相似片段

文本里常见思路有：

- n-gram / shingle
- MinHash / SimHash
- embedding 相似度

图像里常见思路有：

- perceptual hash
- embedding 检索

### 5. 最小代码里每一步在做什么？

完整代码见 [minimal.py](minimal.py)。

```python
def reservoir_sample(items, k, seed=0):
    rng = random.Random(seed)
    result = []
    for idx, item in enumerate(items):
        if idx < k:
            result.append(item)
        else:
            j = rng.randint(0, idx)
            if j < k:
                result[j] = item
    return result
```

这里演示了最常见的在线采样思路之一：

- 数据流很长时，不需要一次全读进内存
- 仍然能近似均匀抽样

再看 near dedup：

```python
def shingle(text, n=3):
    text = normalize_text(text)
    return {text[i : i + n] for i in range(max(1, len(text) - n + 1))}

def jaccard(a, b):
    return len(a & b) / max(1, len(a | b))
```

这里体现的是最小文本近重复判断：

- 先做归一化
- 再切 n-gram
- 最后用 Jaccard 看相似度

## 面试高频问题

### 1. 数据获取为什么不能只看量？

因为分布覆盖、长尾样本、新鲜度和泄漏风险同样决定训练效果。

### 2. exact dedup 和 near dedup 的区别是什么？

exact dedup 去完全重复，near dedup 去几乎重复但不完全相同的样本。

### 3. 去重为什么会影响评测？

因为训练评测泄漏会让离线指标虚高。

### 4. 采样策略为什么要和任务绑定？

因为不同任务关注的分布不同，预训练、垂类微调和 hard case 挖掘的采样重点都不一样。

## 工程关注点

- 去重阈值过严会不会误删
- 跨来源、跨时间窗口怎么做泄漏检查
- 图像、文本、视频是否需要不同 dedup 方案
- hard case 是否在去重后被误伤

## 常见坑点

- 只做 exact dedup，不做 near dedup
- 采样完全按方便程度走，不看目标任务分布
- 忽略训练集和评测集交叉污染
- 去重指标只看召回，不看误删代价

## 面试时怎么讲

一个比较稳的讲法是：

> 数据获取阶段核心是覆盖和分布，不能只看规模。采样要围绕目标任务设计，去重则至少要做 exact dedup 和 near dedup 两层，否则一方面浪费训练预算，另一方面容易引入分布偏置和评测泄漏。
