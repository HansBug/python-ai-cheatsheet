# 质量过滤 / 噪声清洗 面试攻略

## 这是什么？

这是数据工程部分的第二篇，专门讲“数据拿到以后，怎么把明显脏样本清掉”。

如果只用一句话概括：

> 质量过滤的目标，不是把数据越筛越少，而是用规则、模型分数和人工抽检把低价值、高噪声和高风险样本尽量挡在训练集外面。

## 核心机制

### 1. 为什么数据清洗不是可选项？

因为原始数据源通常会混进很多低质量样本，例如：

- 空文本
- 截断文本
- 重复乱码
- 图文不匹配
- 低清、黑屏、损坏图片
- 标签明显错误

这些噪声会直接带来：

- 训练不稳定
- 收敛变慢
- 模型学到错误相关性

### 2. 常见过滤信号有哪些？

最基础可以分成三层：

#### 1. 规则信号

例如：

- 长度阈值
- 字符占比
- 特殊符号比例
- 图像分辨率
- 视频时长

#### 2. 模型信号

例如：

- 语言识别分数
- OCR 置信度
- 图文一致性分数
- 安全分类器分数

#### 3. 人工抽检反馈

规则和模型都不可能完全可靠，所以通常还会留一层：

- 抽样检查
- 误杀回看
- 阈值回调

### 3. 为什么“高质量”不能只看单一分数？

因为单一分数常常有盲区。

例如：

- 文本很长，不代表有信息量
- 图像很清晰，不代表和文本匹配
- teacher model 打分高，不代表没有系统性偏差

更稳的做法通常是：

- 多个质量信号组合
- 分桶分析误杀和漏放
- 对重要 slice 单独设阈值

### 4. 多模态数据清洗为什么更难？

因为它不仅要看单模态质量，还要看跨模态一致性。

例如图文数据里常见问题有：

- 图片正常，文本正常，但两者不匹配
- OCR 文本和图中实际内容冲突
- 视频字幕和画面不同步

所以多模态过滤通常至少要问：

> 每个模态本身是不是干净，以及不同模态之间是不是对得上。

### 5. 最小代码里每一步在做什么？

完整代码见 [minimal.py](minimal.py)。

```python
def score_text_quality(text):
    stripped = text.strip()
    if not stripped:
        return 0.0

    unique_ratio = len(set(stripped)) / len(stripped)
    alpha_ratio = sum(ch.isalnum() for ch in stripped) / len(stripped)
    length_score = min(len(stripped) / 32.0, 1.0)
    return 0.4 * unique_ratio + 0.3 * alpha_ratio + 0.3 * length_score
```

这里用了三个最小信号：

- 信息多样性
- 正常字符占比
- 长度是否过短

再看多模态组合分数：

```python
def score_record(record):
    text_score = score_text_quality(record["text"])
    image_text_score = record.get("image_text_score", 0.5)
    ocr_score = record.get("ocr_score", 0.5)
    return 0.5 * text_score + 0.3 * image_text_score + 0.2 * ocr_score
```

这段代码的重点不是具体权重，而是说明：

> 真实的数据清洗几乎总是多信号融合，不会只靠一个阈值。

## 面试高频问题

### 1. 质量过滤为什么不能只靠规则？

因为规则覆盖有限，很容易漏掉语义级噪声或跨模态错配。

### 2. 为什么模型打分也不能完全信？

因为 teacher model 本身也有偏差，可能系统性误杀某些长尾样本。

### 3. 多模态清洗比单模态难在哪里？

不仅要看每个模态本身，还要看模态之间是否一致。

### 4. 质量阈值怎么定？

通常要结合抽检、误杀率、漏放率和 downstream 效果一起调。

## 工程关注点

- 误杀 hard case 的代价
- teacher score 的偏置和漂移
- 不同来源是否要分源设阈值
- 抽检样本怎么分桶才有代表性
- 过滤后数据分布是否被意外扭曲

## 常见坑点

- 只做规则清洗，不做抽检回看
- 只看平均质量分，不看 slice
- 过滤完数据更“干净”，但任务覆盖被破坏
- 过度依赖 teacher model，忽略偏差放大

## 面试时怎么讲

一个比较稳的讲法是：

> 数据清洗本质上是在做多信号质量判别。规则可以清掉明显脏样本，模型分数可以补语义层过滤，但最终还得靠抽检和 slice 分析去校准阈值，否则很容易把长尾高价值样本误杀掉。
