# 标注体系设计 / 指南编写 面试攻略

## 这是什么？

这是数据工程部分的第三篇，讲的是最容易被低估的一环：

- 标签怎么定义
- 标注任务怎么拆
- 标注指南为什么会直接决定数据质量
- 一致性差时应该怎么处理

如果只用一句话概括：

> 标注不是“把人找来打标签”这么简单，而是先把任务、标签边界、冲突处理和验收标准定义清楚，再让标注系统稳定地产出一致标签。

## 核心机制

### 1. 为什么标注体系设计是算法问题，不只是流程问题？

因为标签定义本身会决定模型学到什么。

如果标签边界模糊，就会带来：

- 标注员理解不一致
- 模型学到混乱监督
- 线上 badcase 难以归因

所以标注体系设计本质上是在回答：

> 任务到底要模型学什么，不学什么。

### 2. 一个标注任务至少要定义什么？

最少要定义四件事：

#### 1. 标签空间

- 有哪些标签
- 是否互斥
- 是否允许多选
- 有没有层级结构

#### 2. 判定规则

- 什么情况打正例
- 什么情况打负例
- 边界样本怎么处理

#### 3. 输入输出格式

- 是单标签、多标签，还是 span / box / polygon
- 是否需要理由、证据或属性字段

#### 4. 验收标准

- 一致率门槛
- 抽检规则
- 冲突仲裁方式

### 3. 为什么指南里一定要写边界 case？

因为真正拉开数据质量差距的，往往不是简单样本，而是：

- 模糊样本
- 冲突样本
- 多意图样本
- 信息不全样本

如果指南只写“标准正例”，不写边界 case，标注员很容易各自发挥。

所以一个靠谱的标注指南通常都会包含：

- 正例
- 反例
- 难例
- 冲突处理规则

### 4. 一致性差时，应该先看什么？

不要一上来怪标注员，先看三件事：

- 标签定义是不是模糊
- 指南有没有覆盖边界样本
- 界面和输入信息是不是不足

如果这些没定义好，光靠培训很难根治问题。

### 5. 最小代码里每一步在做什么？

完整代码见 [minimal.py](minimal.py)。

```python
task = AnnotationTask(
    name="image_safety",
    labels={"safe", "unsafe"},
    multi_label=False,
)
```

这里先把任务规范显式写出来，而不是隐含在文档里。

再看校验逻辑：

```python
def validate_annotation(task, annotation):
    labels = annotation["labels"]
    if not labels:
        return False, "empty labels"
    if not task.multi_label and len(labels) > 1:
        return False, "multi-label not allowed"
    if not set(labels).issubset(task.labels):
        return False, "unknown label"
    return True, "ok"
```

这段代码对应的直觉是：

- 先保证标签格式合法
- 再谈质量和一致性

最后看一致性：

```python
def agreement_rate(annotation_groups):
    matched = sum(len(set(group)) == 1 for group in annotation_groups)
    return matched / max(1, len(annotation_groups))
```

虽然很简化，但已经说明：

> 标注体系不只要能收标签，还要能量化一致性。

## 面试高频问题

### 1. 标注指南最重要的内容是什么？

不是标签名字本身，而是边界定义、冲突规则和验收标准。

### 2. 为什么一致性问题不能只靠培训解决？

因为很多分歧来自任务定义本身不清楚，而不是执行力问题。

### 3. 多标签任务和互斥单标签任务为什么要提前定清楚？

因为这会直接影响标注界面、校验逻辑和模型训练目标。

### 4. 标注质量怎么量化？

至少要看抽检通过率、标注员一致率、仲裁比例和关键 slice 的错误分布。

## 工程关注点

- 标签体系是否可扩展
- 边界样本是否有单独规则
- 多语言、多模态输入时标注信息是否足够
- 仲裁成本和一致性指标如何平衡
- 标注结果如何回流到训练和评测体系

## 常见坑点

- 先启动大规模标注，再补指南
- 标签名字很漂亮，但边界没定义
- 只看平均一致率，不看具体冲突类型
- 标注任务和模型训练目标脱节

## 面试时怎么讲

一个比较稳的讲法是：

> 标注体系设计本质上是在把任务目标形式化。先定义标签空间、边界规则、输入输出和验收标准，再通过校验、一致性统计和仲裁机制保证标签可用。高质量标注不是靠人多，而是靠任务定义清楚、边界 case 写明白。
