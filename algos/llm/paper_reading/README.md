# LLM 模型家族精读索引

## 这是什么？

这里专门放各个主流开源 LLM 家族的 paper reading / 技术报告精读，不和通用机制说明混在一起。

## 收录原则

- 只要属于面试高频、社区主流，且公开了论文、技术报告、系统说明或 model card，就应建立独立条目
- 有正式 paper 的模型按 paper reading 写
- 没有正式 paper、但有技术报告的模型按技术报告精读写
- 同一家族多代模型优先放在同一篇里做纵向比较，改动过大时再单独拆分

## 每篇条目的固定问题

每个模型家族条目都应至少回答下面这些问题：

- 它相对前代或同类模型，核心改了什么？
- 这些改动分别影响训练稳定性、推理成本、长上下文还是效果上限？
- 数据、tokenizer、context length、alignment recipe 有没有关键变化？
- 这些变化里，哪些值得在面试中重点讲，哪些只需要点到为止？
- 如果让你和其他高频开源家族做比较，应该怎么讲？

## 条目模板

建议每个条目包含以下结构：

```markdown
# [模型家族名] 精读

## 这是谁，解决什么问题？
## 架构上改了什么？
## 训练配方和数据上改了什么？
## 推理和部署上有什么影响？
## 和前代 / 同类模型相比该怎么讲？
## 面试高频追问
## 我会怎么口头讲给面试官听
```
